# Sprint 1 — Ядро: Детальний опис

## Що робимо

Створюємо мінімально працездатну версію гри: від структури проєкту до повного ігрового циклу в терміналі. Після цього спрінту гра запускається командою `uv run python -m tictactoe_cli`, відображає дошку через Rich, приймає ходи гравця і відповідає через AI (minimax).

## Як робимо

### Підхід
1. Налаштовуємо проєкт (pyproject.toml, структура пакетів, залежності, pytest, CI + ai-reviewbot)
2. Створюємо генератор символів — окремий модуль, що повертає ASCII-арт матриці для X та O
3. Реалізуємо Rich Renderer як підклас `Renderer` з бібліотеки `tic-tac-toe-3x3`
4. Реалізуємо Human Player як підклас `Player` з обробкою координатного введення
5. Збираємо все в Game Runner, що використовує `TicTacToe` engine

### Тестування
Тести пишуться разом з кожною таскою (test-alongside-development):
- task-01 → `tests/test_smoke.py`
- task-02 → `tests/test_symbols.py`
- task-03 → `tests/test_renderer.py`
- task-04 → `tests/test_input.py`
- task-05 → `tests/test_app.py`

Кожна таска вважається завершеною тільки коли її тести зелені.

### Технічні рішення
- Символи X/O — матриці 17x7 символів із блоків `█`, зберігаються як списки рядків
- Дошка — Rich `Panel` або прямий вивід через `Console`, подвійні Unicode-рамки (`╔═╗║╚╝╦╩╠╣╬`)
- Координати: колонки A/B/C зверху, ряди 1/2/3 зліва
- Введення: рядок формату `[a-c][1-3]`, маппінг на індекс 0-8 через формулу `(row-1)*3 + col`
- Game Runner: інстанціює `TicTacToe(player1, player2, renderer)` і викликає `.play()`

### Залежності між тасками
```
task-01 (структура) → task-02 (символи) → task-03 (рендерер) → task-05 (runner)
                                            ↑
                      task-04 (введення) ───┘
```

## Очікувані результати

1. `uv sync` встановлює всі залежності без помилок
2. `uv run python -m tictactoe_cli` запускає гру
3. В терміналі відображається порожня дошка 3x3 з рамками та координатами
4. Гравець вводить хід (наприклад, `b2`), символ з'являється на дошці
5. AI відповідає ходом через minimax
6. Гра завершується з відображенням результату (перемога/нічия)
7. `ruff check` проходить без помилок
8. `mypy --strict` проходить без помилок
9. `pytest` — smoke test проходить
10. GitHub Actions CI — green на push/PR
11. PR отримує автоматичне AI-ревью від ai-reviewbot
